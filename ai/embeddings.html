Embeddings & Vector Representations

How machines convert language into numbers — and why the numbers carry meaning. The foundation of every modern AI search and RAG system.

▶

The Core Problem

Computers are exceptional at math but cannot read. When you write the word "king", a computer sees raw characters — k, i, n, g. It has no concept that king and queen are related, or that monarch means the same thing. There's no meaning, no relationship, no understanding — just bytes.

So how do modern AI systems understand language? By converting it into math. Specifically, into a list of numbers called a vector. And if you do this conversion correctly — using a trained neural network — words with similar meanings end up with similar numbers.

The goal: represent language as numbers in a way that preserves meaning. Similar meaning → similar numbers.

What is a Vector?

A vector is simply a list of decimal numbers. It could be 3 numbers, 384 numbers, or 3072 numbers. Think of it like GPS coordinates — except instead of pointing to a location on a map, these coordinates point to a location in meaning-space.

# A simplified 6-dimensional vector for "dog" dog_vector = [0.82, -0.14, 0.93, 0.07, 0.61, -0.38] # Real embedding models: 384 to 3072 dimensions # all-MiniLM-L6-v2 → 384 dimensions # text-embedding-3-large → 3072 dimensions

Words and sentences with similar meanings end up with similar vectors — meaning they're close together in this multi-dimensional space. "Dog", "cat", and "wolf" cluster together. "King", "queen", and "monarch" cluster together. The two clusters are far apart from each other.

Why does this work?

The positioning isn't arbitrary. Embedding models are trained on billions of sentences, and they learn which words appear in similar contexts. Words that appear in similar contexts end up with similar vectors. This is the distributional hypothesis — meaning comes from context.

What is an Embedding?

Not every vector is an embedding. An embedding is a specific kind of vector — one produced by a trained neural network that has learned to encode semantic meaning from language.

The difference matters. A random list of numbers is a vector. An embedding is a vector where the numbers represent learned meaning. The network was trained so that semantically similar text produces numerically similar vectors.

Definition

An embedding is a dense, fixed-size vector produced by a neural network that encodes the semantic meaning of text. The closer two embeddings are in vector space, the more similar their meaning.

The pipeline

When you pass text through an embedding model, here's what happens:

from sentence_transformers import SentenceTransformer model = SentenceTransformer('all-MiniLM-L6-v2') # Pass any text → get a 384-dimensional vector back embedding = model.encode("The cat sat on the mat") print(embedding.shape) # (384,) print(embedding[:5]) # [ 0.84, -0.21, 0.67, 0.12, 0.55]

Embedding Model Families

Several providers produce embedding models. The main families to know:

OpenAI

text-embedding-3-small (1536 dimensions) and text-embedding-3-large (3072 dimensions). Paid API, excellent quality, widely used in production. Best quality-to-cost ratio for most use cases.

Sentence Transformers (open source)

all-MiniLM-L6-v2 (384 dimensions) is the most popular open source model. Runs locally, free, fast. This is what you should use for local development and learning. bge-large-en-v1.5 from BAAI consistently tops open source benchmarks.

Cohere

embed-english-v3.0 (1024 dimensions). Strong retrieval performance, multilingual variant available. Paid API.

Google

text-embedding-004 (768 dimensions) via Vertex AI and the Gemini API. Well-integrated into the Google ecosystem.

For learning: start with all-MiniLM-L6-v2 — free, runs locally, no API key needed. For production: evaluate text-embedding-3-small from OpenAI — excellent quality at low cost.

Cosine Similarity — Finding Similar Vectors

Once everything is a vector, "finding similar text" becomes a geometry problem: find vectors that are closest to a given query vector.

The most common metric is cosine similarity. Instead of measuring straight-line distance between vectors, it measures the angle between them. Why angle? Because two sentences can be different word lengths but mean the same thing — they point in the same direction, even if their magnitudes differ.

# Cosine similarity formula # cos(θ) = (A · B) / (||A|| × ||B||) from sklearn.metrics.pairwise import cosine_similarity import numpy as np model = SentenceTransformer('all-MiniLM-L6-v2') sentences = [ "fast cars", "speedy vehicles", # similar meaning "banana recipes", # unrelated ] embeddings = model.encode(sentences) # fast cars vs speedy vehicles → ~0.85 (very similar) # fast cars vs banana recipes → ~0.12 (unrelated) print(cosine_similarity([embeddings[0]], [embeddings[1]]))

Score interpretation: 1.0 = identical meaning · 0.0 = unrelated · -1.0 = opposite meaning.

Semantic Search — Putting it Together

Traditional keyword search matches exact words. Semantic search matches meaning. This is the payoff for everything above.

Query: "How do I fix my broken phone screen?"

A document that says "cracked display repair guide" means the same thing but uses completely different words. Keyword search misses it. Semantic search finds it because both embed to similar vectors.

# Full semantic search pipeline query = "broken phone screen" documents = [ "cracked display repair", "fix shattered LCD panel", "mobile screen replacement guide", "banana bread recipe", ] # 1. Embed query and all documents query_emb = model.encode([query]) doc_embs = model.encode(documents) # 2. Find top-K by cosine similarity scores = cosine_similarity(query_emb, doc_embs)[0] ranked = sorted(zip(scores, documents), reverse=True) # Result: all 3 phone-related docs rank high, banana ranks low

This exact pipeline — embed everything, store vectors, embed query, find nearest neighbours — is the foundation of every RAG system. Day 2 goes deeper on where and how to store millions of these vectors efficiently.

Core concepts

  • A vector is a list of numbers that points to a location in mathematical space
  • An embedding is a vector produced by a neural network that encodes semantic meaning
  • Similar meaning → similar vectors → close together in vector space
  • Embedding models are trained on billions of sentences to learn these relationships

Key metrics

  • Cosine similarity measures angle between vectors — most common for text
  • Score of 1.0 = identical meaning, 0.0 = unrelated, -1.0 = opposite
  • Dot product similarity is faster but requires normalised vectors
  • Euclidean distance also works but less common for text embeddings

Model recommendations

  • Learning/local: all-MiniLM-L6-v2 (free, 384-dim, fast)
  • Production: text-embedding-3-small (OpenAI, best cost/quality)
  • Best OSS quality: bge-large-en-v1.5 (BAAI)
  • Multilingual: Cohere embed-multilingual-v3.0

Common mistakes

  • Using keyword search when you mean to do semantic search
  • Mixing embedding models — query and docs must use the same model
  • Ignoring dimensionality — higher dims ≠ always better
  • Not normalising vectors before using dot product similarity
