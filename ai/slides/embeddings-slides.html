The Problem

Computers can't "read" — they only see numbers

Words like "king", "queen", and "monarch" mean similar things to us. But a computer sees raw characters — no meaning, no relationship.

We need a way to convert language into math — a form computers can compare, calculate, and reason about.

The answer: turn every word or sentence into a list of numbers called a vector.
Vector Representations

A vector is just a list of numbers — with a direction

Think of it as GPS coordinates, but for meaning. Every word or sentence gets a fixed-size list of decimal numbers (e.g. 384 or 1536 dimensions).

Words with similar meanings end up with similar coordinates — close together in this mathematical space.

# "dog" becomes a 6D vector
embedding = [
  0.82, -0.14, 0.93,
  0.07,  0.61, -0.38
]
# Real models: 384 to 3072 dimensions
Embeddings

An embedding is a vector that captures meaning

Not all vectors are embeddings. An embedding is a vector produced by a trained neural network that has learned to encode semantic meaning from millions of text examples.

The network compresses language into a dense, fixed-size space where semantic relationships are preserved mathematically.

An embedding is a vector — but one where the numbers represent learned meaning, not arbitrary values.

Similarity Search

Finding the closest meaning in vector space

Once everything is an embedding, "search" becomes a geometry problem: find the vector closest to the query vector.

The most common metric is Cosine Similarity — it measures the angle between two vectors, not their distance. Two sentences can be different lengths but mean the same thing.

cos(θ) = A · B / (‖A‖ × ‖B‖)
1.0 = identical meaning  ·  0.0 = unrelated  ·  -1.0 = opposite
Semantic Search

Search by meaning, not by keywords

Traditional search matches exact keywords. Semantic search matches meaning.

Query: "How do I fix my broken phone screen?"

Traditional search misses docs that say "cracked display repair" — same meaning, different words. Semantic search finds it.

Semantic search is powered by embedding the query and all documents, then finding nearest neighbours in vector space.
