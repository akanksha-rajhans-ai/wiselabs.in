The Problem

Keyword search is blind to meaning

BM25 matches exact tokens. Same intent, different words — completely missed. This was acceptable in 2005. It breaks every modern AI product.

BM25 doesn't know that "heart attack" and "myocardial infarction" are the same thing. It only sees characters.

How Semantic Search Works

Meaning becomes geometry

Text with similar meaning ends up in the same neighbourhood of vector space. Search becomes: find the vectors closest to the query vector.

The distributional hypothesis: words that appear in similar contexts have similar meanings — and similar vectors.

king − man + woman ≈ queen. Meaning has direction and geometry in vector space.

The Full Pipeline

Offline indexing + Online query

The offline phase runs once (batch). The online phase runs in under 100ms per query. Keep them decoupled — they scale independently.

Approximate Nearest Neighbour

Exact search doesn't scale — ANN does

Scanning every vector at 10M docs = billions of multiplications per query. ANN (HNSW) navigates a graph to find approximate neighbours in milliseconds.

O(n·d)
exact NN complexity
~5ms
HNSW at 10M vectors
~95%
recall vs exact

The 5% you miss are usually the lower-quality results anyway. ANN is always the right choice at scale.

Hybrid Search

BM25 + Semantic, fused with RRF

Neither approach is universally better. Production systems combine both using Reciprocal Rank Fusion.

RRF Formula

score(d) = Σ 1 / (60 + rank(d))

k=60 reduces impact of top ranks · works across incompatible score scales · no normalisation needed

k=60 was found empirically. You never need to tune it in practice — just use the default.

Two-Stage Retrieval

Retrieve fast, rerank precisely

Bi-encoders are fast but approximate. Cross-encoders are slow but accurate. Use both in sequence.

Bi-encoder — embeds query and doc independently. Fast, scalable, used for retrieval (top 50).

Cross-encoder — sees query + doc together. Slow, accurate, used for reranking (50 → 5).

The pattern: retrieve 50 with bi-encoder → rerank to 5 with cross-encoder. This is the standard for any serious retrieval system.

Failure Modes

When semantic search breaks — and the fix

Exact strings needed

Serial numbers, error codes, order IDs, SKUs. Nearest semantic neighbour of "ORA-12541" might be a completely different error.

✓ Fix: Use BM25 or direct lookup for structured identifiers

Out-of-domain vocab

Legal Latin, medical jargon, proprietary internal terms. Model wasn't trained on this vocabulary — embeddings are noise.

✓ Fix: Fine-tune on domain data or use domain-specific model

Very short / ambiguous queries

"Python" — language, snake, or comedy group? One-word queries have no context for the embedding model to work with.

✓ Fix: Query expansion · hybrid search · HyDE

Stale embeddings

Document updated, embedding not re-indexed. Silent failure — no error, just wrong results serving outdated content.

✓ Fix: Re-indexing pipeline triggered on document update
