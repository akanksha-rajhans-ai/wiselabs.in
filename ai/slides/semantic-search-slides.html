The Problem

Keyword search is blind to meaning

BM25 matches exact tokens. Same intent, different words — completely missed. Acceptable in 2005. Breaks every modern AI product.

BM25 doesn't know that "heart attack" and "myocardial infarction" are the same thing. It only sees characters.

Slide 3 of 9 — How It Works
How Semantic Search Works

Meaning becomes geometry

Text with similar meaning ends up in the same neighbourhood of vector space. Search becomes: find the vectors closest to the query vector.

The distributional hypothesis: words appearing in similar contexts have similar meanings — and similar vectors.

king − man + woman ≈ queen. Meaning has direction and geometry in vector space.

Slide 5 of 9 — ANN Search
Approximate Nearest Neighbour

Exact search doesn't scale — ANN does

Scanning every vector at 10M docs = billions of multiplications per query. ANN (HNSW) navigates a graph to find approximate neighbours in milliseconds.

O(n·d)
exact NN
~5ms
HNSW @ 10M
~95%
recall vs exact

The 5% you miss are usually lower-quality results anyway. ANN is always the right choice at production scale.

HOW HNSW WORKS
Layer 2 (sparse) — entry point, long jumps
↓
Layer 1 — medium granularity, greedy search
↓
Layer 0 (dense) — result found 🎯
No full scan — navigates graph in O(log n) time

Slide 6 of 9 — Hybrid Search
Hybrid Search

BM25 + Semantic, fused with RRF

Neither approach wins universally. Production systems combine both using Reciprocal Rank Fusion — no score normalisation needed.

RRF Formula

score(d) = Σ 1 / (60 + rank(d))

k=60 (default) · uses rank positions not raw scores · works across incompatible scales

k=60 was found empirically. You never need to tune it — just use the default.

Slide 7 of 9 — Reranking
Two-Stage Retrieval

Retrieve fast, rerank precisely

Bi-encoders are fast but approximate. Cross-encoders are slow but accurate. Use both in sequence.

Bi-encoder — embeds query and doc independently. Fast, scalable. Used for retrieval (top 50).

Cross-encoder — sees query + doc together. Slow, accurate. Used for reranking (50 → 5).

Pattern: retrieve 50 with bi-encoder → rerank to 5 with cross-encoder. Standard for any serious retrieval system.

Stage 1 — Bi-encoder
All vectors → ANN → top 50 candidates (fast)
↓ 50 candidates
Stage 2 — Cross-encoder
Query + doc pairs → precise relevance → top 5 (accurate)
↓ top 5

Slide 8 of 9 — Failure Modes
Failure Modes

When semantic search breaks — and the fix

Exact strings needed

Serial numbers, error codes, order IDs. Nearest semantic neighbour of "ORA-12541" might be a completely different error.

✓ Fix: BM25 or direct lookup for structured identifiers

Out-of-domain vocab

Legal Latin, medical jargon, internal terms. Model wasn't trained on this — embeddings are noise.

✓ Fix: Fine-tune on domain data or use domain model

Short / ambiguous queries

"Python" — language, snake, or comedy? No context for the embedding model to work with.

✓ Fix: Query expansion · hybrid search · HyDE

Stale embeddings

Document updated, embedding not re-indexed. Silent failure — no error, just outdated results.

✓ Fix: Re-indexing pipeline on document update
